# Changelog

## [2.12.1] - 2026-06-09

Главное: обновление sing-box до 1.14.0-alpha.29; управление встроенным WG-сервером из UI; крупная переработка редакторов туннелей; серия фиксов подписок (стабильность + производительность материализации) и sing-box/прокси.

### Новые возможности

- **Управление встроенным WG-сервером из UI** — настройка внутреннего сервера прямо из программы.
- **vlink: транспорт httpupgrade** — добавлена поддержка httpupgrade.
- **sing-box: копирование proxy-URL** прямо из интерфейса.
  
### Исправления

- **Подписки** — выбор активного члена не переписывает слот; материализация ускорена.
- **Подписки** — не создают NDMS Proxy при выключенном тумблере.
- **Singbox маршрутизатор** — запрет создания циклических urltest/selector с ссылками на себя.
- **Access-policy** — туннели больше не показываются down в политиках после ребута (инвалидация кэша OpypTun).
- **Скачивание конфигурации через AWG-туннель** — DNS запрос через туннель исправлен.
- **Встроенный WG-сервер** — нумерация начинается с 0, а не с 1.
- **ASC** — улучшены генерация синтетической сигнатуры и дизайн обфускации.
- **vlink: транспорт shadowsocks** — исправлена поддержка shadowsocks.

### UI

- **Переработка редакторов туннелей** — унифицированные редакторы, обновлённый интерфейс.
- **Список sing-box туннелей** — корректный empty-state; человеко-понятные мелочи серверов/туннелей.
- **Починены** — тема (возврат заголовка); скролл config-checker; мигание бейджа при refresh.

- **Подписки** — создание не ломает `40-subscriptions.json` (PR [#288](https://github.com/hoaxisr/awg-manager/pull/288)).
- **Мастер sb-router** — выбор туннеля показывает все outbounds (PR [#311](https://github.com/hoaxisr/awg-manager/pull/311)); имя подписки вместо `sub-<hash>` (PR [#274](https://github.com/hoaxisr/awg-manager/pull/274)).

- Человеко-читаемые имена outbound в чипах соединений (PR [#292](https://github.com/hoaxisr/awg-manager/pull/292)); понятная подпись маршрутизации через sing-box (PR [#293](https://github.com/hoaxisr/awg-manager/pull/293)).

- **DoH-апстримы NDMS** — исправлен парсинг (раньше валилась диагностика DNS).
- **Singbox boot-wait** — увеличен до 45s + env override, version probe через sidecar.
- **Singbox validation** — двухпроходная валидация подписки + hard waitForSingbox (PR [#221](https://github.com/hoaxisr/awg-manager/pull/221)).
- **Kernel туннели** —  исправление состояния "запускается".
- **NDMS** — IPv6 allow-ips mask больше не теряется (PR [#216](https://github.com/hoaxisr/awg-manager/pull/216)).
- **Router** — DNS server `type=local` больше не ломает конфигурацию.

### UI

- **Мобильная адаптация** (PR [#214](https://github.com/hoaxisr/awg-manager/pull/214)) — composite outbound, DNS-конфигуратор, header «Интерфейсы политики», таблица правил sing-box, settings/monitoring/diagnostics карточки.
- **HR Neo** на мобильном (PR [#219](https://github.com/hoaxisr/awg-manager/pull/219)).
- **DNS-правила** — зависание после bulk-смены туннеля исправлено (PR [#222](https://github.com/hoaxisr/awg-manager/pull/222)).
- **Theme mode switch** — растянут на desktop.
- **Server header actions** — стабилизированы на мобильном.
- **Update** — рефреш кэшированных pending-обновлений.
- **Develop channel gate** — persist pass.
- **Settings** — селектор служебных загрузок скрыт без sing-box (PR [#215](https://github.com/hoaxisr/awg-manager/pull/215)).

## [2.11.0] - 2026-05-23

Релиз посвящён производительности и стабильности: разговор с роутером ускорился в несколько раз (через объединение запросов и кеши), модуль `awg-proxy` доведён до идентичности с образцом amnezia-kernel, sing-box получил режим без роутерных прокси связок для тех, кто будет использовать sing-box router, появилось безопасное резервное копирование управляемых серверов.

### Производительность

- **NDMS-запросы ускорены** — средняя задержка запросов к роутеру снижена до 50%, число «медленных» обращений (>1 с) упало.
- **Параллельные циклы и кеши** — для всех туннелей теперь запрашивается параллельно, некоторые подсистемы переведены на кешированные данные вместо повторных запросов.
- **Задержка запросов после сохранения изменний в конфигурации роутера** — после `save` даём роутеру таймаут и сразу инвалидируем кеш.
- **Метрики** — периодический итог работы в логе: (сколько запросов схлопнулось в один HTTP-вызов), среднее время ответа на запрос.

### NativeWG / awg-proxy (kmod 1.1.7 → 1.1.9)

- **Самостоятельная XChaCha20-Poly1305 реализация** (PR [#164](https://github.com/hoaxisr/awg-manager/pull/164) автор **@AYastrebov**) — заменён вызов kernel crypto API на встроенную byte-for-byte реализацию.
- **Прозрачный пересчёт MAC2 на cookie-обмене** (PR [#155](https://github.com/hoaxisr/awg-manager/pull/155) автор **@AYastrebov**) — при rate-limit сервер присылает encrypted cookie, дальнейшие handshakes должны нести MAC2 поверх своего MAC1. Без этого fix'а под нагрузкой клиент стабильно ломался после первого `cookie reply`. Спасибо Андрею.
- **S4 transport prefix** — заполняется случайными байтами, как в reference, а не нулями.
- **Жёсткая валидация конфигов** — отклоняется неточный hex publickey и пересекающиеся magic headers.
- **HR Neo policies отделены от common** — теперь в политики созданные HR Neo нельзя добавить устройства, это не требуется для работы этого инструмента.

**Для применения нового модуля **перезагрузка** роутера обязательна.**

### Sing-box: переключатель NDMS-прокси

- **Опция «Создавать NDMS Proxy для sing-box»**  — в настройках появился toggle, позволяющий выключить создание NDMS Proxy-объектов для sing-box туннелей. Зачем: некоторые сценарии маршрутизации не требуют NDMS-проксей, и их создание добавляло лишнюю нагрузку на роутер при каждом save. 
  - На странице туннеля бейдж показывает «via sing-box», iperf и другие тесты отключёны.
- **Дедупликация туннелей по fingerprint** — повторные вызовы с теми же outbound-параметрами больше не создают дубли в конфиге.

### Sing-box router: bypass-порты и фильтрация

- **Bypass пресеты по портам** — на странице sing-box роутера в разделе «Дополнительно» теперь можно выбрать порты, трафик по которым пойдёт мимо sing-box маршрутизатора:
  - L2TP / IPsec / IKE / NTP / NetBIOS / SMB — типичные локально-используемые UDP-протоколы, которые иначе подхватывались sing-box'ом и ломались;
  - Дополнительные UDP/TCP порты вводятся вручную, валидируются (порт 0 отклоняется), сортируются и стабильно реконсилируются;
- **Filtering mode**  — добавлен режим фильтрации в sing-box; правила можно генерировать на лету (`generating-srs`).
- **Inline наборы правил как .srs файлы** — inline-наборы правил больше не интерпретируются на каждый запрос; компилируются в `.srs` и переиспользуются. На странице правил появилось каскадное удаление: при удалении rule-set'а очищаются все ссылки на него в зависимостях.
- **JRS/inline ruleset парсер** — несколько итераций улучшений: корректная обработка инлайнового списка при редактировании rule set'ов, лучше UX модалок, цветовая подсветка JSON-схемы.
- **DNS** — улучшения отображения и UX.

### Резервное копирование управляемых серверов

- **Полный backup управляемых серверов с приватными ключами**  — на странице «Серверы» появились кнопки Export/Import:
  - При создании сервера приватный ключ теперь сохраняется в storage; для legacy-серверов автоматически подтягивает ключи при первом старте программы;
  - ASC-обфускация проходит валидацию и верификацию перед записью.
- **Persisted selection** — выбранный сервер сохраняется между перезагрузками страницы (localStorage), кнопки Export/Import в одном стиле с туннелями.


### NativeWG — стабилизация

- **Issue #136: handshake-fail на replace-config** (PR [#167](https://github.com/hoaxisr/awg-manager/pull/167)) — при замене конфига туннеля приватный ключ и DNS не синхронизировались с интерфейсом, и сервер начинал отказывать в handshake. Теперь sync делается явно. Добавлен regression-тест на `replace-config peer-no` (PR [#163](https://github.com/hoaxisr/awg-manager/pull/163)).
- **Убрана петля ExternalRestart для NWG-туннелей** (PR [#171](https://github.com/hoaxisr/awg-manager/pull/171)) — в редких случаях orchestrator уходил в бесконечный цикл рестарта NWG-туннеля. Триггер устранён. Но роутер по своей логике еще может административно выключить туннель если посчитает что он не работает. Работа в этом направлении еще будет продолжена.

### Единое логирование

- **Полная унификация на ScopedLogger** (PR [#170](https://github.com/hoaxisr/awg-manager/pull/170)) — все модули пишут в один поток через с подгруппами (`orchestrator`, `kmod`, `storage`, `http` и т. д.).

### Изменения работы

- **Программа больше не пытается запустить hrneo , если HR Neo не установлен** (PR [#175](https://github.com/hoaxisr/awg-manager/pull/175)).
- **Geodata: запрет правки внешних файлов** — геофайлы, привязанные к внешним источникам (vernette/v2ray-rules-dat и др.), больше не редактируются вручную. Под капотом — переработана работа со списками (geofile adoption для HR Neo, routed geo downloads, защита от записи пустых конфигов).
- **Frontend в Go-бинарнике** — статические файлы теперь упакованы прямо в исполняемый файл, отдельная директория не нужна.
- **OPKG storage**: исправлено отображение свободного места при установке OPKG во внутренюю память.

### Диагностика

- **Обогащённые и санитизированные отчёты** — отчёты диагностики собирают больше системного контекста, при этом чувствительные данные (приватные ключи, токены) фильтруются.
- **Чувствительные хосты в логах sing-box скрываются** — IP-адреса и доменные имена в журнале sing-box обезличиваются (порт сохраняется). Можно безопасно скидывать кусок лога в чат поддержки.
- **Мульти-фильтры в журнале** — фильтрация записей сразу по нескольким уровням и источникам; добавлен переключатель показа таймстампов.
- **Логи в отчёте — только warn/error** — раньше отчёт распухал от info-уровня.
- **Журнал: точная пауза при прокрутке вверх + экспорт** — лог не «убегает», когда листаешь историю.
- **About-tab TXT export** — экспортируется с локальным временем.
- **Кнопка скачивания отчёта убрана из лога** — отчёт качается со страницы Диагностика.

#### NativeWG реализация через awg_proxy.ko — серия фиксов 

На ряде серверов NativeWG туннель через `awg_proxy.ko` либо не устанавливал handshake вовсе, либо принимал первый handshake но через ~60 секунд сервер прекращал отвечать. Серия инкрементальных правок (kmod версии 1.1.3 → 1.1.6) приводит к байт-идентичности с reference `amneziawg-linux-kernel-module`:

- **MAC1 пересчёт на всех 4 путях handshake** (PR [#138](https://github.com/hoaxisr/awg-manager/pull/138) автор **@AYastrebov**) — `awg_proxy.ko` ранее пересчитывал MAC1 только на 2 из 4 типов handshake-пакетов (outbound init и inbound response), пропуская outbound response и inbound init. Когда сервер инициировал rekey или клиент отвечал на init, MAC1 не совпадал и пакет дропался. Спасибо Андрею за чистый референс-кросс-чек и точечный фикс. (kmod 1.1.3), именно этот PR вернул веру в возможность починить функционал.

- **HANDSHAKE_DSCP=0x88 на init/response** — reference выставляет AF41 DSCP на handshake-пакетах. Мы отправляли с TOS=0. Промежуточные узлы по пути дропали (особенно для CPS с DNS-shape контентом). (kmod 1.1.4)

- **DF=0 (Don't Fragment не установлен)** - приведено в соответствии с reference.

- **Удалён pre-ARP 0-byte UDP "probe"** — наша собственная оптимизация для прогрева ARP оказалась первым видимым пакетом от нашего источника к серверу. (kmod 1.1.6)

- **Inter-packet jitter в handshake-цикле** — наш kthread отправлял CPS+junks+init back-to-back за <1мс. Reference (через workqueue) даёт ~7-8мс на цикл. Добавлен `usleep_range(1500,2500)` между sendmsg-ами. (kmod 1.1.6)

### Под капотом (способ реализации)

- Новый mangle-фикс `AWGM-DNS-NOPOLICY` — `internal/singbox/router/iptables.go` + `lanbridges.go`. На каждом LAN-bridge с настроенным NDMS hotspot catch-all'ом эмитим правило `-m mark --mark 0x0 -m pkttype unicast --dport 53 -j MARK --set-mark <bridge-NDMS-catch-all-mark>`. Mark передаётся per-bridge — динамически читается из `_NDM_HOTSPOT_PREROUTING_MANGL` (не хардкод), что делает фикс корректным независимо от того какая именно policy назначена default в NDMS.
- `DiscoverLANBridges(ctx)` — пересечение `(NDMS hotspot mangle catch-all)` ∩ `(real Linux bridges in /sys/class/net/*/bridge)`. Это набор bridges где (a) device МОЖЕТ оказаться с mark=0 (через MAC-override) и (b) у NDMS уже есть готовая `_NDM_HOTSPOT_DNSREDIR` инфраструктура для этого mark'a. Исключает WireGuard tunnels и не-bridge интерфейсы.
- Пустой discovery (роутер без hotspot-конфига) → skip с warn'ом, install продолжается без DNS-NOPOLICY. Регрессия невозможна.
- `Reconcile` детектит изменение LAN-bridges (added/removed bridge, mark переназначен) → re-install iptables.
- netfilter.d hook расширен comment-tag scrub'ом — после NDMS reload правила DNS-NOPOLICY восстанавливаются без duplicate-accumulation.

### Новые возможности

- **Раздел «Мониторинг»** — новая страница `/monitoring` с матрицей `target × tunnel`. Показывает latency пингов через каждый работающий туннель к набору целей: Cloudflare DNS (1.1.1.1), Google DNS (8.8.8.8), Quad9 DNS (9.9.9.9), плюс динамически добавляются цели, настроенные как pingcheck-target у конкретных туннелей.
  - Ячейки окрашены по порогам: **<100ms** зелёный, **100–250ms** жёлтый, **>250ms** красный, **failed** серый.
  - Бейдж «★» на ячейке показывает, что эта цель влияет на рестарт туннеля (совпадает с настроенным pingcheck-target).
  - Клик по ячейке открывает боковую панель с графиком latency за последний час (60 точек, обновление каждые 60 сек), статистикой (avg/min/max/loss%) и последними 10 замерами.
  - Сверху страницы — статус-полоса с 4 показателями: всего туннелей, up, failed, средняя latency.
  - Auto-refresh через SSE: матрица обновляется в реальном времени без перезагрузки страницы.
- **Раздел «Диагностика»** переработан в terminal-style ленту логов с боковой панелью «здоровья».
  - **Лента логов**: новые записи появляются сверху с blinking cursor, авто-пауза при прокрутке вниз, buffer-counter «+N новых ↑» для возврата к LIVE. Click на `[scope]` или `[LEVEL]` фильтрует по нему. Right-click открывает контекстное меню (скопировать строку, скопировать сообщение, фильтр по scope, фильтр по уровню). Ctrl+F фокусирует поиск внутри страницы.
  - **Кумулятивные фильтры по уровню**: выбор `INFO+` показывает info, warn, error. Раньше работала строгая фильтрация по одному уровню.
  - **Боковая панель «Здоровье»** содержит запуск проверок системы (с подробной настройкой режима + opt-in restart-цикла), результаты последнего запуска с раскрываемым списком тестов, и DNS-проверку (свёрнутая по умолчанию).
  - Подвкладка **«Соединения»** сохранена с косметическим обновлением.
- **Новые тесты диагностики**:
  - **Расхождение времени** — сравнивает локальное время с эталонным сервером (HTTPS Date header). Расхождение >300с = FAIL, >60с = WARN. Wireguard handshake чувствителен к skew, частая причина «туннель не поднимается».
  - **rp_filter check** — для каждого работающего туннеля читает `/proc/sys/net/ipv4/conf/<iface>/rp_filter`. Strict-mode (=1) блокирует обратный трафик через VPN — частая причина «туннель up, но интернета нет».
- **Расширение страниц** — `Мониторинг` и `Диагностика` теперь занимают всю ширину экрана, без пустых полей по бокам. Свойство `width` контейнера: `narrow` (960px) для форм, `wide` (1440px) дефолт, `full` (100%) для terminal/grid.

### Изменения работы

  - Остановит туннель
  - Заменит Interface и Peer
  - Сохранит все настройки: имя, ISP-привязку, мониторинг, правила маршрутизации
  - Запустит туннель обратно

  DNS Route Failover — резервирование DNS-маршрутов

  Если в правиле DNS-маршрутизации указано несколько туннелей (primary + backup), AWG Manager
   теперь сам переключает трафик на резервный туннель, когда primary падает (определяется
  через PingCheck). При восстановлении primary трафик автоматически возвращается. В UI
  показывается toast-уведомление о переключении.

  Это обходной путь для бага в прошивках Keenetic, где встроенный механизм fallback в
  dns-proxy route работает ненадёжно.

  Комментарии в IP-маршрутах

  В правилах IP-маршрутизации теперь можно добавлять комментарии к подсетям через синтаксис
  1.2.3.4/32 !ASTelegram. Комментарии:
  - Парсятся при импорте .bat/.txt файлов (после строки маршрута)
  - Передаются в роутер через RCI (видны в нативном WebUI Keenetic)
  - Отображаются на карточке правила в виде тегов

  Идеально для импортов с группировкой по ASN или сервисам.

  Замена опросов на Server-Sent Events (SSE)

  Полная переработка обмена данными между frontend и backend:
  - Все обновления (статус туннелей, трафик, логи, мониторинг) приходят в реальном времени
  через одно SSE соединение
  - Убрано опрашивание раз в 5–30 секунд → меньше нагрузка на роутер, мгновенные обновления в
   UI
  - Графики трафика обновляются без задержек
  - При перезапуске сервиса страница автоматически перезагружается

  Orchestrator — единый мозг управления туннелями

  Полностью переписан слой управления жизненным циклом туннелей. Теперь все события

  orchestrator'ом, который принимает решения декларативно через таблицу состояний. Это
  устранило целый класс race condition'ов и сделало поведение предсказуемым.

## [2.5.0] - 2026-03-16

  AWG Manager 2.5.0

  NativeWG — нативные WireGuard-туннели через ядро Keenetic

  **Главное нововведение версии. Туннели теперь могут работать через встроенный
  WireGuard-модуль Keenetic (NativeWG) вместо userspace amneziawg-go. NDMS управляет
  интерфейсом напрямую — стабильнее, быстрее, переживает перезагрузки.**

На версии ОС Кинетик 5.1+ пока есть проблемы с NativeWG. 

  - Полный lifecycle NativeWG-туннелей: создание, старт, стоп, удаление, обновление через
  NDMS
  - Выбор backend (kernel/nativewg) при импорте конфигурации
  - Proxy-режим: awg_proxy.ko обеспечивает AWG-обфускацию поверх нативного WireGuard
  - Выбор ISP-интерфейса для NativeWG (connect via)
  - Поддержка ping-check мониторинга для NativeWG-туннелей
  - Поддержка системных туннелей в статических маршрутах и политиках

  Управляемый WireGuard-сервер

  Создание и управление собственным WireGuard VPN-сервером прямо из интерфейса AWG Manager.

  - Создание сервера с автоматическим выбором свободного интерфейса
  - NAT включён по умолчанию
  - Управление клиентами: добавление, удаление, включение/отключение
  - Индивидуальные DNS-настройки для каждого клиента
  - Генерация .conf файлов и QR-кодов для клиентов
  - Настройка ASC-параметров обфускации (junk, padding, headers, signatures)
  - Захват signature-пакетов по домену
  - Статистика пиров в реальном времени (LED, трафик, handshake)

  Страница серверов

  - Просмотр всех WireGuard-серверов (управляемых и системных) на одной странице
  - Встроенный Wireguard VPN Server отображается в read-only режиме
  - Возможность перемещать системные интерфейсы между туннелями и серверами
  - Экспорт .conf для пиров (не-встроенных серверов)

  **Удаление userspace backend

  - Amneziawg-go (userspace) полностью удалён — только kernel и nativewg
  - Уменьшен размер IPK-пакета
  - Упрощена архитектура**

  Улучшения интерфейса

  - Скрытие системных туннелей с главной страницы
  - Мониторинг (ping-check) переработан: индивидуальные настройки для каждого туннеля
  - Корректное отображение трафика клиентов (зеркальные ↓↑)

  Исправления

  - Корректная очистка при удалении программы (managed server, endpoint routes для nativewg)
  - Миграция NDMS-клиента на RCI API (меньше спама в логах роутера)
  - Поддержка KN-3611 (Hopper DSL)
